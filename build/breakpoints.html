Breakpoints

The string interpolation method of creating breakpoints in Foundation 5 has been replaced with a new breakpoint() mixin.

.element {
  font-size: 1.5rem;

  @include breakpoint(medium) {
    font-size: 2rem;
  }
}

Using Sass

Variables

NameTypeDefault ValueDescription
$breakpoints Map ( small: rem-calc(0), medium: rem-calc(640), large: rem-calc(1200), xlarge: rem-calc(1440), xxlarge: rem-calc(1920), )

A list of named breakpoints. You can use these with the breakpoint() mixin to quickly create media queries.
$breakpoint-classes List (small medium large)

All of the names in this list will be output as classes in your CSS, like small-12, medium-6, and so on. Each value in this list must also be in the $breakpoints map.

Mixins

breakpoint

@mixin breakpoint($val) { }

Wraps a media query around the content you put inside the mixin. This mixin accepts a number of values:

  • If a string is passed, the mixin will look for it in the $breakpoints map, and use a media query there.
  • If a pixel value is passed, it will be converted to an em value using $rem-base.
  • If a rem value is passed, the unit will be changed to em.
  • If an em value is passed, the value will be used as-is.

NameTypeDefault ValueDescription
$val Keyword|Number

Breakpoint name or px/em/rem value to process.

each-breakpoint

@mixin each-breakpoint($class, $omit) { }

Prefixes selector $class with breakpoint keywords, allowing you to create a batch of breakpoint classes with one chunk of code. If you want to skip a breakpoint (like small, because mobile first and all that), add values to the $omit parameter.

NameTypeDefault ValueDescription
$class string

Class to prefix with the breakpoint name and a hyphen.
$omit list

Named breakpoints to skip. No class will be added with breakpoints in this list.
